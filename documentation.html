canDo()

canDo( canvasElement, canDoConfiguration )


Returns: A canDo wrapped 2d rendering context.

Description: Wraps a canvas element and adds methods and properties for building simple animations.

canDo( canvasElement, configuration )


canvasElement  An HTMLCanvasElement.

configuration A set of key/value pairs that configures the canDo animation.

When a canvas element is wrapped in canDo all of the native methods and properties of the 2dRenderingContext are still available.

configuration

A set of key/value pairs that configures the canDo animation.

duration: int [ Optional ]
Default: 1000

The duration of the timeline in 1/1000 seconds. The actual of duration of the 
animation can also be changed adjusting the playback speed.


frameRate: int [ Optional ]
Default: 30

The targeted refresh rate in frames per second. canDo will not exceed this rate,
but may render fewer frames per second if there is to much load on the system.


setTime: float (min: 0.0, max: 1.0) [ Optional ]
Default: 0.0

Sets the current playback position on the timeline. Valid values are 
0.0 (the beginning) through 1.0 (the end).


mode: string [ Optional ]
Default: ''

The playback mode for the animation. Valid values are:

'' (empty string): Play through the animation until an endpoint is reached and stop.

'loop': When the endpoint is reached, rewind and play again. When the playback 
speed is a negative value (playing backwards) the animation will loop back to the 
end when the beginning is reached and continue playing.


wait: boolean [ Optional ]
Default: true

When set to true the animation will not play until all images passed in have been 
loaded into memory. When set to false the animation will play regardless of the 
state of the images.


splash: boolean [ Optional ]
Default: true

When set to true the first frame of the animation will be rendered wether the 
animation is playing or not.
