{ canDoConfiguration }

canDo( canvasElement, canDoConfiguration )

Description: A set of key/value pairs that configure the canDo animation.

{
	paint: function() {},
	[, duration: int ] 		// Default: 1000
	[, framerate: int ] 		// Default: 30
	[, setTime: float ] 		// Default: 0.0
	[, playbackSpeed: float ] 	// Default: 1.0
	[, mode: string ] 		// Default: ''
	[, easing: string ] 		// Default: 'linear'
	[, wait: boolean ] 		// Default: true
	[, splash: boolean ] 		// Default: true
	[, cuePoints: { } ] 		// Default: {}
	[, events: { } ] 		// Default: {}
	[, images: { } ] 		// Default: {}
}

When a canvas element is wrapped in canDo all of the native methods and properties of the 2dRenderingContext are still available.

Example

var translate = canDo(document.getElementById('translate'), {
    frameRate: 60,
    duration: 3000,
    easing: 'easeInOutQuad',
    paint: function() {
        this.identity();
        this.clearRect(0, 0, 320, 320);
        this.canDo('translate', [ 
        	{params: [0, 0],cuePoint: 0},
        	{params: [110, 110],cuePoint: 1.0}
        ]);
        drawJSLogo(this);
    }
});

Methods

Properties

configuration

duration: int [ Optional ]
Default: 1000

The duration of the timeline in 1/1000 seconds. The actual of duration of the 
animation can also be changed adjusting the playback speed.


frameRate: int [ Optional ]
Default: 30

The targeted refresh rate in frames per second. canDo will not exceed this rate,
but may render fewer frames per second if there is too much load on the system.


setTime: float (min: 0.0, max: 1.0) [ Optional ]
Default: 0.0

Sets the current playback position on the timeline. Valid values are 
0.0 (the beginning) through 1.0 (the end).


playbackSpeed: float [ Optional ]
Default: 1.0

Sets the current playback speed. A value of 1.0 will play at the normal speed 
based on the duration of the animation. Values less than 0.0 will play the
animation backwards.


mode: string [ Optional ]
Default: ''

The playback mode for the animation. Valid values are:

	'' (empty string): Play through the animation until an endpoint is 
	reached and stop.

	'loop': When the endpoint is reached, rewind and play again. 
	When the playback speed is a negative value (playing backwards) the 
	animation will loop back to the end when the beginning is reached and 
	continue playing.


easing: string [ Optional ]
Default: 'linear'

The easing function to use on the timeline. Putting the easing function 
here eases the playback head of the animation and is effectively the same as 
putting the same easing function on every call to canDo or canSet, but is much 
more effecient. Valid values are:

	'linear': No easing
	
	'swing': A popular function that eases both in and out

wait: boolean [ Optional ]
Default: true

When set to true the animation will not play until all images passed in have been 
loaded into memory. When set to false the animation will play regardless of the 
state of the images.


splash: boolean [ Optional ]
Default: true

When set to true the first frame of the animation will be rendered wether the 
animation is playing or not.


cuePoints: A set of key/value pairs referencing 
points on the timeline by name [ Optional ]
Default: {}

cuePoints are great when working with an art director who
"knows what they like when they see it". You can move a keyframe
on multiple elements by changing one value in your configuration file.


events: A set of key/value pairs defining events 
that should be bound to the canvas element [ Optional ]
Default: {}

Basic event handling for you canvas element


images: A set of key/value pairs declaring images that are used in the 
animation [ Optional ]
Default: {}

canDo will automatically ]create image objects and optionally preload all the 
images used in your animation.


paint: The callback function that is used to render a frame of the 
animation [ Required ]

This is the meat in your animation hamburger.
